IO 控制案例

该教程会实现控制吸盘吸取和释放的插件。

该插件的工作流程如下：

创建插件​

# 需要使用 node v20 及以上版本
dpt create

在初始化插件时，需要提供

  • 插件的名字，必填，当前文件夹下不得有同名文件夹
  • 插件的描述，非必填，可以后续在配置文件中进行修改
  • 插件的版本号，默认是 1-0-0
  • 机械臂控制器的IP地址，默认是 192.168.5.1，可在配置文件中修改
$ dpt create

 grip
? Please input plugin description A plugin demo for gripper
? Please input plugin version 1-0-0
? Please input device IP 192.168.5.1

机械臂&末端控制​

机械臂和末端的控制逻辑主要在 userApi.lua 文件中进行编写。 针对该插件的场景，实际需要对外提供三个函数：

  • 吸取函数
  • 释放函数
  • 获取状态函数

针对这三个函数，我们需要依次编写 userApi.lua、httpApi.lua和对应的 UI 文件。

    • 编辑页面的交互事件

      import { Button } from '@dobot-plus/components'
      import { useState } from 'react'
      import { requestToGrip, requestToRelease } from '../../http/api'
      import { DobotPlusApp } from '@dobot/components/DobotPlusApp'

      enum STATUS {
          GRIPPING = 0,
          LEISURE = 1,
          ERROR = 2
      }

      const STATUS_LABEL = {
          [STATUS.GRIPPING]: '吸取中',
          [STATUS.LEISURE]: '空闲',
          [STATUS.ERROR]: '异常'
      }

      function App() {
          const [status, setStatus] = useState(STATUS.LEISURE)

          function handleGrip() {
              requestToGrip()
          }

          function handleRelease() {
              requestToRelease()
          }

          const receiveMessageHandle = (topic: string, message: Buffer) => {
              const data = JSON.parse(message.toString())
              setStatus(data.status)
          }

          return (
              <DobotPlusApp
              useMqtt={true}
              onMessage={receiveMessageHandle}>
                  <div className="app">
                      <h1>{STATUS_LABEL[status]}</h1>
                      <Button onClick={handleGrip}>吸取</Button>
                      <Button onClick={handleRelease}>释放</Button>
                  </div>
              </DobotPlusApp>
          )
      }

      export default App

调试和验证​

调试插件指令可进行以下两种情形的开发工作：

  • 仅调试页面
  • 连接真机进行调试
dpt dev

在执行上述命令时，命令行会提示开发者是否连接真机进行测试

$ dpt dev
? Debug lua on real device? Yes
? Please check the device IP: 192.168.5.1 and port: 22001 (y/n)

开发者需要确定：

  • 控制器的真实IP是否正确，默认是 192.168.5.1
  • 控制器服务端口是否正确，默认是 22001
  • SFTP 服务相关配置是否正确

上述配置的详细信息请查看 dpt.json 配置文件

{
  "ip": "192.168.5.1",                              // 控制器 IP
  "ftp": {
    "port": 22,                                     // SFTP 服务端口
    "username": "root",                             // 非特殊情况，请勿修改该用户名
    "password": "9f128f8d2342a690a435ee0c6cdcf916", // 非特殊情况，请勿修改该密钥
    "remotePath": "/dobot"                          
  },
  "port": 22001,                                    // 控制器端口
  "pluginPort": 22100
}

机械臂快捷键​

用户可在Dobot+ 菜单中选择 末端快捷键 菜单 从下来的插件列表中选择需要的插件

该功能对应着 httpAPI.lua 中的如下代码

httpModule.OnRegistHotKey = function() 
	return {press = {"demoMethod1", "demoMethod2"}, longPress = {"demoMethod3"}}
end

其中快捷键和函数的对应关系如下：

菜单名称对应关系
快捷键1demoMethod1
快捷键2demoMethod2
长按demoMethod3

构建插件​

在完成插件的开发、调试、优化后，可执行最终的构建工作，执行

dpt build

在程序顺利执行完毕后，当前文件夹下会出现 dist 文件夹和 output 文件夹。

  • dist 文件夹中存放着本次构建后的插件代码，用于开发者检查构建结果
  • output 文件夹存放着压缩后的 zip 文件，文件名格式为 <插件名>-<版本号>.zip，该文件为实际在客户端导入使用的的插件。
