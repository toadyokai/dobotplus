Development in GitHub Codespace

Dobot+ plugin development also supports cloud development in GitHub Codespaces.
Developers can create a copy of the Dobot+ plugin template and develop in the cloud on their own branch, without needing to set up a local development environment.

  • Dobot+ Plugin Template Repository: https://github.com/toadyokai/dobot-plus-template.git
  • Dobot+ Plugin Template NPM Package: https://www.npmjs.com/package/@dobot-plus/template

Developers can fork the template repository to their own repository, or manually download the Dobot+ plugin template, create a local repository, and push it to GitHub.

Codespaces is a paid service.

Individual developers have a free monthly quota of 15GB storage and 120 hours of compute time.

Environment Initialization​

Under the Code menu on the page, users can create a Codespaces workspace for the current repository.

After the workspace is created, the browser will display the VS Code coding interface.

The first time you enter, a pop-up in the lower right corner will recommend VS Code extensions for the Dobot+ template project. These extensions are helpful for improving development efficiency and are recommended for installation.

In the VS Code terminal, run pnpm i to install the npm dependencies for the current template project.

Since Dobot+ plugin development requires the Dobot+ SDK CLI, you also need to run npm install @dobot-plus/cli --global in the terminal, as shown below:

After completing the above steps, the basic cloud development environment is set up.

You can run the following commands in the terminal to check if the tool versions meet the requirements:

node -v # Check node version

pnpm -v # Check pnpm version

dpt -v  # Check dpt CLI version

Development and Debugging​

After setting up the basic environment, run the following command in the terminal:

dpt dev

This is the same as running

dpt dev

locally. Make sure that browser pop-up notifications are enabled to allow the preview page to open and redirect automatically.

Once the command executes successfully, the browser will automatically open the plugin preview interface, as shown below:

You can debug the plugin interface interactions and styles as needed.

If the preview page does not open automatically, you can manually construct the preview URL based on the current Codespaces URL and open it in the browser. The default preview port is 8080.

Modify the part before .github.dev by adding -8080 to manually open the preview link in the browser.

Plugin Packaging​

Run the plugin build command in the Codespaces terminal:

dpt build

After the build is complete, open the output folder from the file menu on the left, right-click the zip file inside, and select Download. After a short wait, the packaged plugin will be downloaded to your browser's download directory.

Continue Development​

Using Codespaces requires an internet connection. After a development session, if the Codespaces project is not deleted, it will be preserved. You can click the corresponding Codespaces to re-enter and continue development.
